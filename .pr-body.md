## Summary
Adds full **username** support to the authentication system so users can sign in with **email or username** and new users get a required, unique username.

## Changes

### 1. Add New User modal
- Required **Username** field after Full Name
- Rules: unique, case-insensitive, no spaces; letters, numbers, underscores (2-50 chars)
- Helper text: Username must be unique
- Validation and uniqueness check on create

### 2. Login
- Single field accepts **email or username**
- Label: Email or Username; placeholder: Enter email or username
- If input contains @ then treat as email; else treat as username (case-insensitive)

### 3. Backend
- **Login API**: Lookup by email or username (same user record)
- **Create user**: Validate and enforce unique username (stored lowercase)
- **Update user**: Optional username with uniqueness check

### 4. Database
- Added `username` column (nullable, UNIQUE) for backward compatibility
- Migration: 20260209000000_add_username_to_users
- Existing users keep logging in with email only

### 5. UI and i18n
- Add/Edit User forms and User Details show username
- Login page and translations (EN/AR) updated
- Seed and init-admin/create_admin/setup-cli set username for new users

## Acceptance
- [x] Login with email
- [x] Login with username
- [x] Username required and unique for new users
- [x] Existing users unchanged
- [x] No breaking changes
